次のJavaScriptのコードを
フローチャート図にしてください。
```
let i = 1;
while (i <= 10) {
  console.log(i);
  i = i + 1;
}
```

---

このコードは、

* 変数 `i` を 1 で初期化
* `i <= 10` の間くり返す
* `i` を表示
* `i` を 1 増やす

という流れですね。

## このコードに対応するフローチャート（テキスト表現）

```
   ┌─────────┐
   │  開始   │
   └────┬────┘
        │
        ▼
   ┌─────────┐
   │ i = 1   │
   └────┬────┘
        │
        ▼
   ┌──────────────┐
   │ i <= 10 ?    │◀────────────┐
   └────┬─────┬───┘             │
        │Yes  │No                │
        ▼     ▼                  │
   ┌─────────┐  ┌─────────┐     │
   │ i を表示 │  │  終了   │     │
   └────┬────┘  └─────────┘     │
        │                        │
        ▼                        │
   ┌─────────┐                   │
   │ i = i+1 │                   │
   └────┬────┘                   │
        │                        │
        └───────────────戻る─────┘
```

---

## ポイント（初心者向けに説明すると）

* ◇（ひし形）は **条件分岐**
* □（四角）は **処理**
* 矢印が「処理の流れ」
* 上に戻る矢印があると **繰り返し（ループ）**

---

もしよろしければ、

* PowerPoint用の図
* PDF教材用のきれいな図
* draw.io用XML
* Mermaid記法

のどれか形式を指定していただければ、その形式で作ります。

---

条件分岐（ひし形）ではなく、
ループ（台形）を使った図がほしいのです。

---

なるほど、「条件分岐（ひし形）」ではなく
**ループ記号（台形）を使ったフローチャート**ですね。

※流儀によって多少記号は異なりますが、
「ループの開始条件を台形でまとめる」書き方があります。

## このコードに対応する図（台形ループ記号使用）

```
      ┌─────────┐
      │  開始   │
      └────┬────┘
           │
           ▼
      ┌─────────┐
      │ i = 1   │
      └────┬────┘
           │
           ▼
     ╱────────────────╲
    ╱   i <= 10 の間   ╲   ← ループ（台形）
    ╲────────────────╱
           │
           ▼
      ┌─────────┐
      │ i を表示 │
      └────┬────┘
           │
           ▼
      ┌─────────┐
      │ i = i+1 │
      └────┬────┘
           │
           └───────────┐
                       │
            （条件が真なら上へ戻る）
                       │
                       ▼
                  ┌─────────┐
                  │  終了   │
                  └─────────┘
```

---

## ポイント

この書き方では：

* ひし形で Yes/No を分けない
* 「〜の間」という**ループ構造を1つの構造として表す**
* 構造化プログラミング寄りの表現
