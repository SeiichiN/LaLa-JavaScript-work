jqueryを使って画像のプリロードを以下のように書いてみました。

```
$(function(){
  const images = [
    'images/image1.jpg',
    'images/image2.jpg',
    'images/image3.jpg',
    'images/image4.jpg',
    'images/image5.jpg',
  ];

  images.forEach(function(item) {
    preloadImage(item);
  });

  function preloadImage(path) {
    $('img').attr('src', path);
  }
}
```

特に18行目の ```$('img')``` が気になります。
これで img要素の生成ができているでしょうか？

---

いい質問です ??

ご提示のコードを見てみると、18行目の

```js
$('img').attr('src', path);
```

は **「すでに存在する `<img>` 要素を取得して、その `src` を書き換える」** だけの処理です。
つまり **新しい `<img>` 要素を生成するわけではありません**。

---

### なぜこれだと「プリロード」にならないのか？

* もし HTML に `<img>` が1つもなければ、`$('img')` は空の jQuery オブジェクトなので、実際には何も起きません。
* 仮に `<img>` が1つあったとしても、ループのたびに同じ `<img>` に `src` を上書きしてしまうので、最後の画像しか読み込まれません。

---

### 正しく「プリロード」する方法

プリロードの目的は「ユーザーに見せない状態でブラウザに画像を読み込ませてキャッシュに置いておく」ことです。
そのためには **新しい `Image` オブジェクトを生成する**のが一般的です。

```js
$(function() {
  const images = [
    'images/image1.jpg',
    'images/image2.jpg',
    'images/image3.jpg',
    'images/image4.jpg',
    'images/image5.jpg',
  ];

  images.forEach(function(path) {
    preloadImage(path);
  });

  function preloadImage(path) {
    const img = new Image();
    img.src = path;
    // jQueryを使うなら $('<img>') としてDOMに追加もできる
  }
});
```

これなら DOM に追加しなくてもブラウザが画像を読み込んでキャッシュしてくれます。

---

### jQueryを活かしたいなら

もしどうしても jQuery を使いたいなら、非表示で DOM に追加しておく方法もあります：

```js
function preloadImage(path) {
  $('<img>').attr('src', path).css('display', 'none').appendTo('body');
}
```

---

? まとめると：

* `$('img')` は **既存の `<img>` を取得するだけ**。生成はしていない。
* プリロードには `new Image()` を使うか、非表示の `<img>` を作って DOM に追加するのが正しいやり方。
